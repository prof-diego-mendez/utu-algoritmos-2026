Módulo 05 · Estructuras de Control

Estructura repetitiva
WHILE en Java

Las estructuras repetitivas permiten que un programa ejecute un bloque de código varias veces, mientras se cumpla una condición. Son esenciales para procesar colecciones, acumular resultados y automatizar tareas.

¿Qué es un bucle?

Un bucle (o ciclo) repite un bloque de instrucciones mientras una condición booleana sea verdadera. Sin ellos, para sumar 1000 números necesitaríamos 1000 líneas de código. Con un bucle, lo resolvemos en 4.

①
WHILE
Evalúa la condición antes de ejecutar el cuerpo. Puede ejecutarse 0 veces si la condición es false desde el principio.
②
DO-WHILE
Evalúa la condición después. El cuerpo se ejecuta al menos una vez, sin importar la condición.
③
break / continue
Controlan el flujo dentro del bucle: break lo termina anticipadamente, continue salta la iteración actual.

Bucle WHILE

El cuerpo se ejecuta mientras la condición sea verdadera. Antes de cada iteración Java evalúa la condición; si es false, el bucle termina y la ejecución continúa tras la llave de cierre.

Sintaxis general

while (condiciónBooleana) { cuerpo }

La condición debe cambiar en algún momento dentro del cuerpo; de lo contrario el bucle nunca termina (bucle infinito).

Las 4 partes esenciales del WHILE

Todo bucle WHILE correcto tiene cuatro partes bien definidas. Si falta alguna el programa tendrá un error o entrará en un bucle infinito.

1

Inicialización

Declará y asigná el valor inicial a la variable de control antes del while. Sin ella el compilador no sabe con qué valor arrancar.

2

Condición

Expresión booleana evaluada antes de cada iteración. Cuando es false, el bucle termina. Usá operadores relacionales y lógicos.

3

Cuerpo

Las instrucciones que se repiten. Puede ser una sola línea o un bloque complejo. Aquí se realiza el trabajo útil del bucle.

4

Actualización

Modifica la variable de control para que la condición eventualmente sea false. Sin ella el bucle nunca termina.

⚠ Bucle infinito: el error más temido

Un bucle infinito ocurre cuando la condición nunca se vuelve false. Causas comunes: olvidarse de actualizar la variable (i++), actualizar en dirección equivocada (i-- cuando debería ser i++), o usar != con un valor que nunca se alcanza exactamente.

Módulo 05 · DO-WHILE

Bucle
DO-WHILE

La variante del WHILE que garantiza al menos una ejecución del cuerpo. La condición se evalúa al final de cada iteración, no al principio.

Sintaxis y ejemplo

La diferencia clave con WHILE: el cuerpo se ejecuta primero y la condición se evalúa después. Esto hace que el cuerpo siempre se ejecute al menos una vez, incluso si la condición es false desde el principio.

⚠ No olvides el punto y coma

La sintaxis DO-WHILE requiere un ; después del while(condición). Es el único caso en Java donde un while termina con punto y coma. Olvidarlo es un error de compilación frecuente.

WHILE vs DO-WHILE: comparación lado a lado

Ingresá un valor inicial para x. El bucle repite mientras x < 5. Observá cómo cambia el comportamiento según el tipo de bucle.

Comparador WHILE / DO-WHILE
WHILE
int x = 3;
while (x < 5) {
  System.out.println(x);
  x++;
}
// salida (— iteraciones)
—
DO-WHILE
int x = 3;
do {
  System.out.println(x);
  x++;
} while (x < 5);
// salida (— iteraciones)

¿Cuándo elegir DO-WHILE?

Cuando necesitás que el cuerpo se ejecute al menos una vez antes de verificar la condición. El caso más clásico es la validación de entrada del usuario: siempre tenés que pedirle el dato al menos una vez, y luego repetir si no es válido. También es ideal para menús interactivos.

Módulo 05 · Técnicas Avanzadas

Técnicas avanzadas
con bucles

Control de flujo dentro de bucles con break y continue, bucles anidados, el patrón acumulador, y los errores más comunes a evitar.

Instrucción break

break termina el bucle inmediatamente, sin esperar a que la condición sea false. La ejecución continúa en la primera instrucción después del bucle. Es útil cuando encontrás lo que buscabas y no tiene sentido seguir iterando.

Instrucción continue

continue salta el resto del cuerpo de la iteración actual y va directamente a evaluar la condición para la siguiente. El bucle no termina; solo omite parte del trabajo en esa vuelta.

⚠ continue y la actualización de la variable

Si usás continue, asegurate de que la actualización de la variable de control esté antes del continue. Si está después, el continue la saltará en algunas iteraciones y el bucle puede volverse infinito.

Bucles anidados

Un bucle puede contener otro bucle en su cuerpo. Por cada iteración del externo, el interno se ejecuta completo. Son útiles para procesar tablas, matrices o generar patrones.

Clave de los bucles anidados: reiniciar la variable interna

La variable del bucle interno (j) debe reiniciarse dentro del bucle externo, justo antes del bucle interno. Si se declara fuera, la segunda vuelta del externo encontrará j ya en su valor final y el interno no ejecutará.

Patrón acumulador

El acumulador es una variable que reúne un resultado progresivamente a través de las iteraciones. Hay tres sabores principales según la operación:

+=
Suma acumulada
Valor inicial: 0 (neutro de la suma). Usado para sumar series, calcular promedios o totales.
*=
Producto acumulado
Valor inicial: 1 (neutro de la multiplicación). Usado para factoriales, potencias y combinaciones.
++
Contador
Valor inicial: 0. Cuenta cuántos elementos cumplen una condición dentro del bucle.

Errores comunes — incorrecto vs. correcto

✗ Bucle infinito — falta actualización
int i = 1;
while (i <= 10) {
  System.out.println(i);
  // olvidé i++
}
// i nunca cambia → infinito
✓ Correcto — actualización presente
int i = 1;
while (i <= 10) {
  System.out.println(i);
  i++; // avanza hacia el fin
}
✗ continue antes de la actualización
int i = 0;
while (i < 10) {
  if (i % 2 == 0) continue;
  System.out.println(i);
  i++; // nunca llega acá con i par
}
✓ Actualización antes del continue
int i = 0;
while (i < 10) {
  i++; // actualizar primero
  if (i % 2 == 0) continue;
  System.out.println(i);
}
✗ Punto y coma accidental en WHILE
int i = 0;
while (i < 5); // ← ; vacía el cuerpo
{
  System.out.println(i);
  i++;
}
// El while no tiene cuerpo → infinito
✓ Sin punto y coma espurio
int i = 0;
while (i < 5) {
  System.out.println(i);
  i++;
}
// La llave abre el cuerpo real
✗ Neutro incorrecto en acumulador
// Queremos calcular 5!
int fact = 0; // neutro incorrecto
int n = 5;
while (n > 0) {
  fact *= n;
  n--;
}
// fact = 0 (cualquier n * 0 = 0)
✓ Neutro correcto para multiplicación
int fact = 1; // neutro de la multiplicación
int n = 5;
while (n > 0) {
  fact *= n;
  n--;
}
// fact = 120 (correcto)

Buenas prácticas

①

Elegí la condición de salida más clara

Preferí i < 10 o i <= 9 sobre i != 10. Con !=, si la variable saltara el valor exacto (por suma de más de 1), el bucle sería infinito.

②

Poné un límite de seguridad en bucles de entrada

Cuando el bucle depende de entrada del usuario, agregá un contador máximo de intentos para evitar que un bug o entrada maliciosa congele el programa.

③

Declaré la variable del bucle interno dentro del externo

Declarar int j dentro del bucle externo asegura que se reinicia en cada vuelta. Declararla fuera es un error frecuente que hace que el interno no ejecute a partir de la segunda iteración.

④

Usá break con comentario que explique por qué

Un break sin contexto es difícil de leer. Agregá un comentario breve: break; // encontramos el primer divisor. Esto evita confusión sobre por qué el bucle puede terminar antes de la condición principal.

⑤

Limitá el anidamiento a 2 niveles

Más de dos niveles de bucles anidados hace el código difícil de leer y depurar. Si necesitás más profundidad, extraé el bucle interno a un método separado.
