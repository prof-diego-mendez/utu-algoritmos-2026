Módulo 01 · Fundamentos de Java

Tipos de datos primitivos
en Java

Java es un lenguaje fuertemente tipado: toda variable debe declararse con un tipo antes de usarse. Los tipos primitivos son los bloques de construcción más básicos del lenguaje.

¿Qué es un tipo de dato?

Un tipo de dato define qué clase de valor puede almacenar una variable, cuántos bytes de memoria ocupa y qué operaciones se pueden realizar sobre él. En Java, cada variable tiene exactamente un tipo, que no cambia durante la ejecución.

Declaración en Java

La sintaxis general es: tipo nombreVariable = valorInicial;

Ejemplo: int edad = 18; — se reserva memoria, se nombra la variable y se asigna un valor.

Los 8 tipos primitivos

Java define exactamente 8 tipos primitivos. No son objetos: son valores almacenados directamente en memoria (stack).

byte 8 bits · 1 byte
−128 a 127
Entero muy pequeño. Útil para ahorrar memoria en arreglos grandes.
byte b = 100;
short 16 bits · 2 bytes
−32 768 a 32 767
Entero corto. Poco común en código moderno.
short s = 1000;
int 32 bits · 4 bytes
−2 147 483 648 a 2 147 483 647
El tipo entero más usado. Valor predeterminado para literales numéricos enteros.
int edad = 18;
long 64 bits · 8 bytes
±9.2 × 10¹⁸
Entero muy grande. El literal lleva sufijo L.
long pob = 8000000000L;
float 32 bits · 4 bytes
±3.4 × 10³⁸ (~7 dígitos)
Decimal de simple precisión. El literal lleva sufijo f.
float pi = 3.14f;
double 64 bits · 8 bytes
±1.7 × 10³⁰⁸ (~15 dígitos)
Decimal de doble precisión. El más usado para cálculos. Es el tipo predeterminado para literales decimales.
double nota = 9.75;
char 16 bits · 2 bytes
'\u0000' a '\uFFFF' (Unicode)
Un único carácter Unicode. Se escribe entre comillas simples.
char letra = 'A';
boolean 1 bit (lógico)
true · false
Solo dos valores posibles. Esencial para condiciones y estructuras de control.
boolean activo = true;

⚠ Overflow: cuando un número no cabe

Si asignás un valor fuera del rango del tipo, Java produce un overflow: el número "da la vuelta". Por ejemplo, si byte b = 127 y hacés b++, el resultado es −128. Java no lanza error, simplemente el valor se desborda silenciosamente.

Variables y constantes

Reglas y convenciones para nombrar variables

Java distingue reglas obligatorias (el compilador las rechaza si no se cumplen) de convenciones recomendadas (el compilador las acepta, pero el equipo las exige para mantener código legible).

⚙ Reglas obligatorias
  • Solo letras, dígitos, $ y _
  • El primer carácter no puede ser un dígito
  • Sin espacios ni caracteres especiales (-, @, !…)
  • No puede ser una palabra reservada (int, class, return…)
  • Es case-sensitive: nota ≠ Nota ≠ NOTA
  • No tiene límite de longitud

División entera vs. división real

En Java, 7 / 2 = 3 (división entera, sin decimales) porque ambos operandos son int. Para obtener 3.5, al menos uno debe ser double: 7.0 / 2 o (double) 7 / 2.

Módulo 02 · Tipos de Referencia

Tipos no primitivos
(de referencia)

A diferencia de los primitivos, los tipos de referencia almacenan una dirección de memoria (referencia) que apunta al objeto real. Son más flexibles y potentes, pero tienen un costo de memoria mayor.

Primitivo vs. Referencia

Tipo Primitivo
  • Almacena el valor directamente
  • Vive en el stack
  • No puede ser null
  • No tiene métodos propios
  • Comparación con ==
int x = 5; // valor: 5
Tipo de Referencia
  • Almacena una referencia al objeto
  • El objeto vive en el heap
  • Puede ser null
  • Tiene métodos y atributos
  • Comparación con .equals()
String s = "Hola"; // ref → heap

String — La clase más usada

Un String es una secuencia inmutable de caracteres. En Java, es un objeto de la clase java.lang.String, disponible sin importar nada.

Métodos esenciales de String

.length()
Devuelve la cantidad de caracteres.
"Hola".length() // 4
.toUpperCase()
Convierte todo a mayúsculas.
"hola".toUpperCase() // "HOLA"
.toLowerCase()
Convierte todo a minúsculas.
"HOLA".toLowerCase() // "hola"
.charAt(i)
Devuelve el carácter en posición i (base 0).
"Java".charAt(0) // 'J'
.substring(a, b)
Extrae caracteres desde a hasta b (sin incluir b).
"Hola".substring(1,3) // "ol"
.contains(str)
Devuelve true si contiene la cadena indicada.
"Java".contains("av") // true
.replace(a, b)
Reemplaza todas las ocurrencias de a por b.
"aabbaa".replace("a","x") // "xxbbxx"
.trim()
Elimina espacios al inicio y al final.
" hola ".trim() // "hola"
.equals(str)
Compara contenido (no referencia). Siempre usar esto, nunca ==.
s1.equals(s2) // true/false
.equalsIgnoreCase(str)
Igual que equals(), pero ignora mayúsculas y minúsculas.
"HOLA".equalsIgnoreCase("hola") // true
.isEmpty()
Devuelve true si la cadena no tiene caracteres.
"".isEmpty() // true
.indexOf(str)
Devuelve el índice de la primera ocurrencia de str, o -1 si no está.
"Java".indexOf("v") // 2

Autoboxing y Unboxing

Java convierte automáticamente entre primitivos y wrappers. Esa conversión automática se llama autoboxing (primitivo → objeto) y unboxing (objeto → primitivo). Desde Java 5, esto es transparente para el programador.

Módulo 03 · Casting y Conversiones

Conversión
entre tipos de datos

En Java, convertir datos de un tipo a otro es una operación frecuente. Según el nivel de riesgo, puede ser automática o requerir intervención explícita del programador.

Conversión implícita (Widening)

Java convierte automáticamente un tipo "más pequeño" a uno "más grande" sin pérdida de información. No requiere ninguna sintaxis especial.

byte
→
short
→
int
→
long
→
float
→
double

Conversión explícita — Casting (Narrowing)

Cuando convertís de un tipo "más grande" a uno "más pequeño" puede haber pérdida de datos. Debés indicarlo explícitamente con (tipo) para asumir la responsabilidad.

⚠ Riesgo de precisión

Al hacer casting de double a int, la parte decimal se descarta (no se redondea). Por ejemplo: (int) 9.99 devuelve 9, no 10.

Conversión entre String y número

Para procesar entradas del usuario (que siempre llegan como String) necesitás convertirlas a número.

Operaciones mixtas y promoción de tipos

1

Si un operando es double

El otro operando se convierte automáticamente a double. El resultado es double.

2

Si un operando es float

El otro se convierte a float. El resultado es float.

3

Si un operando es long

El otro se convierte a long. El resultado es long.

4

Si ambos son int o menores

Ambos se promueven a int. El resultado es int. Nunca byte ni short.

Concatenación con String

El operador + con String

Cuando un operando de + es String, Java convierte el otro a texto automáticamente. Esto puede ser fuente de errores si se mezclan sumas y concatenaciones en la misma expresión.
