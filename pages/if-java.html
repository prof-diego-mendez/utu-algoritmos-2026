Módulo 04 · Estructuras de Control

Estructura selectiva
IF en Java

Las estructuras selectivas permiten que un programa tome decisiones: ejecutar cierto código solo si se cumple una condición. Son la base del pensamiento lógico en programación.

¿Qué es una estructura selectiva?

En programación, una estructura selectiva (o condicional) evalúa una expresión booleana y, según su resultado (true o false), ejecuta uno u otro bloque de código. Sin ellas, todos los programas solo podrían seguir una secuencia fija.

①
IF simple
Ejecuta un bloque solo si la condición es verdadera. Si es falsa, lo salta.
②
IF-ELSE
Dos caminos posibles: uno para verdadero y otro para falso. Siempre se toma exactamente uno.
③
IF-ELSE-IF
Múltiples condiciones encadenadas. Se ejecuta la primera rama que sea verdadera.

IF simple

La forma más básica. El bloque entre llaves se ejecuta solo si la condición es verdadera. Si es falsa, el programa continúa normalmente.

Sintaxis general

if (condiciónBooleana) { bloqueDeCódigo }

La condición debe ser una expresión que devuelva true o false. Las llaves { } delimitan el bloque que se ejecuta condicionalmente.

IF-ELSE

Agrega una rama alternativa: si la condición es verdadera se ejecuta el primer bloque; si es falsa, el segundo. Siempre se ejecuta exactamente uno de los dos.

IF-ELSE-IF encadenado

Para evaluar múltiples condiciones en orden. Java evalúa de arriba hacia abajo y ejecuta solo la primera rama verdadera; las demás se saltan por completo.

⚠ Orden importa

En un IF-ELSE-IF encadenado, las condiciones deben estar ordenadas de más específica a más general. Por ejemplo, para calificaciones: primero nota >= 9 (más restrictivo) y al final el caso else (más general).

Módulo 04 · Operadores de Condición

Operadores
de comparación y lógica

Los operadores relacionales producen valores true o false. Los operadores lógicos combinan condiciones. Juntos forman las expresiones booleanas que controlan el flujo del programa.

Operadores relacionales

Comparan dos valores y devuelven un boolean. Se usan en las condiciones de los bloques if.

> Mayor que a > b
< Menor que a < b
>= Mayor o igual a >= b
<= Menor o igual a <= b
== Igual a a == b
!= Distinto de a != b

⚠ El error más costoso: = versus ==

a = b es una asignación: guarda el valor de b en a. a == b es una comparación: devuelve true si son iguales. Usarlos equivocados causa errores de compilación (Java) o bugs silenciosos (C/C++). Regla: dentro de un if() siempre va ==.

Operador Significado Ejemplo (a=10, b=5) Resultado
> Mayor que 10 > 5 true
< Menor que 10 < 5 false
>= Mayor o igual 10 >= 10 true
<= Menor o igual 5 <= 5 true
== Igual a 10 == 5 false
!= Distinto de 10 != 5 true

Operadores lógicos

Combinan múltiples condiciones booleanas en una sola expresión. Son esenciales para condiciones complejas.

&&
AND — Y lógico
Verdadero solo si ambas condiciones son verdaderas. Si la primera es false, no evalúa la segunda (short-circuit).
||
OR — O lógico
Verdadero si al menos una condición es verdadera. Si la primera es true, no evalúa la segunda (short-circuit).
!
NOT — Negación
Invierte el valor booleano. !true devuelve false, y !false devuelve true.

Precedencia de operadores

Cuando combinás varios operadores en una expresión, Java aplica un orden de precedencia. Cuanto más alto, se evalúa antes:

1

! — NOT (mayor precedencia)

Se evalúa primero. !a && b equivale a (!a) && b.

2

&& — AND

Más precedencia que OR. a || b && c equivale a a || (b && c).

3

|| — OR (menor precedencia)

Se evalúa al final si no hay paréntesis. Usá paréntesis explícitos para mayor claridad.

Consejo: usá paréntesis siempre que combines && y ||

Aunque la precedencia está definida, expresiones como a || b && !c son difíciles de leer. Escribir a || (b && !c) es más claro y evita bugs futuros. El compilador acepta ambas, pero el equipo agradece la segunda.

Módulo 04 · Formas Avanzadas

Formas avanzadas
del IF en Java

Más allá del IF básico: el operador ternario para expresiones compactas, IF anidados para lógica en capas, comparación correcta de Strings y los errores más comunes a evitar.

Operador ternario ? :

Es una forma compacta de escribir un IF-ELSE cuando el objetivo es asignar un valor según una condición. Tiene exactamente tres partes: condición, valor si es verdadero, valor si es falso.

¿Cuándo usar ternario vs IF-ELSE?

El ternario es ideal para asignaciones simples de una línea donde la lógica es obvia. Preferí IF-ELSE cuando hay múltiples instrucciones, cuando se necesita depurar fácilmente, o cuando la condición es compleja. El código legible siempre gana a la brevedad.

IF anidados

      
Un IF puede contener otro IF dentro de su bloque. Esto permite verificar condiciones en etapas, dando mensajes de error más específicos que con &&.


      


      

        
IF anidado vs. operador &&

        
Ambos pueden expresar lógica compuesta, pero con diferente granularidad:

        
    
          
  • IF anidado: permite saber qué condición específica falló → mensajes más útiles al usuario.
    • 
          
  • &&: más compacto, mejor cuando el mensaje de error no necesita distinguir qué falló.
    • 
        

      


      
      
Comparación de Strings: el error más frecuente

      
Las cadenas de texto son objetos en Java. El operador == compara referencias (si apuntan al mismo objeto en memoria), no el contenido. Para comparar el contenido siempre usá .equals().


      


      
      
Errores comunes — incorrecto vs. correcto


      


        

          

            
✗ Incorrecto

            

              if (edad = 18) {

                ...

              }

              // Error: = es asignación, no comparación
            

          

          

            
✓ Correcto

            

              if (edad == 18) {

                ...

              }

              // == compara el valor
            

          

        


        

          

            
✗ Incorrecto

            

              String s = "hola";

              if (s == "hola") {

                ...

              }

              // Compara referencias, puede fallar
            

          

          

            
✓ Correcto

            

              String s = "hola";

              if (s.equals("hola")) {

                ...

              }

              // .equals() compara el contenido
            

          

        


        

          

            
✗ Incorrecto (riesgo NullPointerException)

            

              String s = null;

              if (s.length() > 0 && s != null) {

                ...

              }

              // .length() lanza error antes de verificar null
            

          

          

            
✓ Correcto (null primero)

            

              String s = null;

              if (s != null && s.length() > 0) {

                ...

              }

              // Short-circuit: si null, no evalúa .length()
            

          

        


        

          

            
✗ Pobre (condición innecesaria)

            

              boolean activo = true;

              if (activo == true) {

                ...

              }

              // == true es redundante con boolean
            

          

          

            
✓ Mejor (más limpio)

            

              boolean activo = true;

              if (activo) {

                ...

              }

              // La variable ya ES boolean
            

          

        


      


      
      
Buenas prácticas


      

        

          
①

          

            
Siempre usá llaves { }

            
Aunque Java permite omitirlas para una sola instrucción, usarlas siempre evita errores futuros cuando agregás más líneas al bloque.

          

        

        

          
②

          

            
Verificá null antes de usar métodos

            
Aprovechá el short-circuit de &&: escribe if (variable != null && variable.metodo()) para evitar NullPointerException.

          

        

        

          
③

          

            
Preferí condiciones positivas

            
Expresiones como if (esActivo) son más legibles que if (!esInactivo). La negación doble complica la lectura.

          

        

        

          
④

          

            
Simplificá condiciones redundantes en IF-ELSE-IF

            
Una vez que entrás a un else if, el rango anterior ya fue descartado. No necesitás repetir el límite inferior del rango.

          

        

        

          
⑤

          

            
Limitá el anidamiento a 2-3 niveles

            
Si tu código tiene muchos IF dentro de IF, considerá simplificarlo con operadores lógicos o restructurando con return anticipado (early return).
